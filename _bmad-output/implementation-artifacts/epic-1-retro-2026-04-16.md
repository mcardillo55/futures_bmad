## Challenges & Growth Areas

### Pattern 1: Edge Case Validation Gaps (5 of 7 stories)

The "never panic, never produce garbage" contract was almost violated repeatedly through boundary condition oversights:

- **Story 1.2:** UnixNanos::to_datetime panics on large u64 values
- **Story 1.3:** Crossed book (best_bid > best_ask) passes is_tradeable
- **Story 1.4:** u32 overflow in Position::apply_fill, zero-size fill creates inconsistent state, zero-quantity orders pass validation, BracketOrder skips component validation
- **Story 1.6:** NaN/Infinity bypass config validation, unparseable dates silently pass, future effective_date not rejected
- **Story 1.7:** cancel_order not recording, PartialFill identical to Fill

**Root cause:** No systematic boundary condition verification during implementation. Review was the only safety net.

### Pattern 2: Missing Derives for Testability (3 stories)

- Bar missing PartialEq/Eq (Story 1.2)
- SignalSnapshot missing PartialEq (Story 1.5)

**Root cause:** Downstream test usage not considered during type definition.

### Pattern 3: Incomplete Mock Behavior (Story 1.7)

MockBrokerAdapter's PartialFill was identical to Fill, and cancel_order discarded the order_id. The mock was incomplete in areas that matter for Epic 7 replay testing.

---

## Key Insights & Learnings

1. **Code review is necessary but insufficient as a sole safety net** — a trading system needs defense in depth. 22 catches in review means 22 things that survived implementation undetected.
2. **Edge case validation is systemic, not incidental** — appearing in 5/7 stories means it's a process gap, not individual oversight.
3. **Property tests are more valuable than unit tests for contract verification** — they found classes of bugs, not just specific instances.
4. **Deferred items must be assessed against the *next* epic's needs** — the 3 UnixNanos/FixedPrice deferrals were acceptable in Epic 1's context but become critical path blockers for Epic 2.

## Technical Debt

### Critical (Must resolve before Epic 2)

1. **UnixNanos::from silently zeroes out-of-range dates** — `timestamp_nanos_opt()` returns None for far-future dates; `unwrap_or(0)` produces epoch zero
2. **UnixNanos::from wraps negative timestamps** — negative i64 from pre-epoch DateTime cast to u64 via `as` wraps to large values
3. **FixedPrice::from_f64 accepts NaN/Infinity** — non-finite f64 values pass through without validation

### Low Priority (Defer to Epic 8)

4. **ConfigValidationError missing std::error::Error impl** — needed for anyhow integration
