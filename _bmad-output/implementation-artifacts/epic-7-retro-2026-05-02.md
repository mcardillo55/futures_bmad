## Successes & Strengths

1. **Three-tier determinism model executed cleanly (7.2).** The split — Tier-1 raw `i64` quarter-ticks (bit-identical), Tier-2 ε=1e-10 for f64 signal values, Tier-3 variant equality for `RegimeState` — is exactly the right shape. Tier-1 captures fixed-point P&L as raw `i64` to avoid `FixedPrice`'s lossy f64 serde representation entirely. The `signal_reset_required` test runs scenarios A (no-reset, expect divergence) AND B (reset, expect match) on the same orchestrator, demonstrating that `Signal::reset()` is genuinely load-bearing — VPIN's bucket leakage is concrete and divergence is reliably caught. Insta snapshot wiring is correct, the snapshot file is small enough to review (3090 lines), and the TOML-driven test config makes the regression mechanism inspectable.

2. **`JournalSender::with_source` source-tag stamping is well-engineered (7.4).** Single `source_override` field rewrites the `source` column on every `TradeEvent` and `OrderStateChange` record at send time. Schema migration is genuinely idempotent (`PRAGMA table_info` guard around `ALTER TABLE` + `CREATE INDEX IF NOT EXISTS`), the column index pair (`idx_trade_events_source`, `idx_order_states_source`) supports the per-source `WHERE` filter without table scan, and FIFO P&L pairing is consistent with the `replay::TradeStats` convention. Default `TradeSource::Live` and `'live'` SQL DEFAULT give belt-and-suspenders fail-closed semantics for backwards-compat — at the cost of B-1 (see Challenges).

3. **Pre-epic cleanup pattern matured into a routine.** Third epic in a row preceded by a focused cleanup commit closing the prior epic's deferred items: Epic 5 → `32f6078` (D-1/D-2/D-3); Epic 6 → `ba8ef22` (A-1/A-2/A-3); Epic 7 implicitly → committed three small items (`pending_transitions` → counter, `last_transition_time` doc, `compute_atr` `.abs()`) before Story 7.1 landed. The pattern is now reflexive.

4. **Hot-path discipline maintained.** Replay loop (7.1) is a tight while-let with three drain calls per iteration, no per-event syscalls beyond SPSC ops. Paper orchestrator (7.3) uses `try_send` non-blocking journal forwarding (canonical pattern from 5.1 / 5.6). `#![deny(unsafe_code)]` on most new modules; clippy clean across the workspace.

5. **Test density.** ~80 new tests with explicit boundary cases: monotonicity guard (7.1 `non_monotonic_timestamp_does_not_advance_clock_backwards`), three-tier determinism (7.2 `replay_produces_identical_results`, `signal_reset_required`, `snapshot_interval_comparison`, `replay_determinism_snapshot`), single-config-knob round-trip (7.3 `shipped_paper_config_routes_to_paper_mode`), source-tag isolation (7.4 `paper_and_live_trades_differ_only_in_source_tag`, `multi_source_isolation_holds_under_interleaved_writes`), 500-trade scale gate (7.4 `readiness_500_trades_completes_with_finite_values`).

6. **Architect-level seams shipped right-shaped, even where the wiring stopped short.** `MarketDataFeed` trait (7.3) is the right abstraction for an orchestrator-level data-source seam. `JournalSender::with_source` (7.4) is the right abstraction for source-tagging without touching every record-emission site. `ExpectedPositionSource` (Epic 6 cleanup) is the right abstraction for the strategy → anomaly producer wire. The seams are correctly placed even when the production-side connection is deferred to Epic 8 — which is the *good* version of the parallel-orchestrator pattern (see P-9 for the bad version).

## Challenges & Growth Areas

### THE STRUCTURAL THEME — Three parallel orchestrators, each skipping the production `EventLoop`

This is the single dominant pattern across all four reviews and deserves its own treatment.

Replay (7.1), Paper (7.3), and the (future) Live orchestrator each run their own private hot-loop that calls `apply_market_event` on a bare `OrderBook` and skips:

- `BufferMonitor` (pressure tracking)
- `StaleDataDetector` (stale-quote watchdog)
- `SequenceGapDetector` (gap detection)
- `DataQualityGate` (data-quality breaker integration)
- `EventWindowManager` (event-window restrictions)
- `CircuitBreakers` (anomaly checks, position-tracker plumbing, per-tick safety)
- `PositionTracker` (broker reconciliation seam)
- `SignalPipeline` (OBI / VPIN / microprice computation)
- `RegimeDetector` + `RegimeOrchestrator` (regime classification, strategy enable/disable)
- `OrderManager` / `BracketManager` (order state machine, journal `TradeEvent` emission)

The cascading evidence:

- **7-1 S-1:** Replay's `run()` claims to flow through "the same `EventLoop` (no replay-specific branching)" — but `event_loop.rs` is untouched and the orchestrator runs its own loop. The module docstring is aspirational, not descriptive.
- **7-3 S-1, S-2:** Paper orchestrator owns `MockBrokerAdapter` ornamentally. Orders bypass `BrokerAdapter::submit_order` and feed directly into `MockFillSimulator`. AC2 ("all circuit breakers / risk / regime function normally") is unverifiable because none of those subsystems are constructed by the orchestrator. The `circuit_breakers_trip_identically_in_paper_mode` test proves the breaker code works in isolation, NOT that the orchestrator integrates it.
- **7-4 B-1 (the blocking finding):** `OrderManager` and `BracketManager` are constructed only in `#[cfg(test)]` blocks. Production `engine --paper` writes ZERO `source = paper` trade rows. The 9 integration tests pass because they manually call `paper_sender.send(JournalEvent::TradeEvent(...))` — the `record_fill` helper at `paper_trade_recording.rs:63-77` is the canonical antipattern: integration tests stitching the data the production wiring doesn't yet provide.
- **Spec-vs-implementation reframe.** 7.4 Task 2.2 specified "stamp at BrokerAdapter level"; the implementation correctly puts it on `JournalSender` because of 7.3 S-1 (orders don't traverse `BrokerAdapter`). The dev notes claimed spec-conforming when the spec was structurally wrong — the honest framing is "the spec presumed an architecture that doesn't exist in V1; the seam moved to where the data actually flows." P-7 (Epic 6 retro: decision-needed handling discipline) didn't fire because this wasn't an auto-deferred decision; it was a silent reframe in the dev notes.

**Net consequence for Story 8.2:** This is no longer "wire one missing piece." Story 8.2's scope today encompasses ~7 OPEN deferred items + parallel-orchestrator unification + paper-OrderManager wiring + reconciliation race + partial-entry SL/TP escalation + `submit_order` consultation of `permits_trading()`. That is too much for one story. See P-9 below.

### `MarketDataFeed::next_event() -> Option<MarketEvent>` semantic gap

`Option::None` conflates "idle right now" and "stream ended." For replay's bounded Parquet source the conflation happens to work (the `None` from a depleted source legitimately ends the loop). For Story 8.2's live Rithmic feed, the FIRST momentary quiet period would terminate `pump_until_idle`. The dev's claim in 7.3 dev notes — "the only change required for Epic 8 is providing a `RithmicMarketDataFeed: MarketDataFeed` impl and swapping the `let feed = ` line" — is structurally wrong. The trait shape itself needs to grow before live data can wire in cleanly.

This is the single load-bearing pre-Epic-8 cleanup item. See B-1 below.

### `decision_id = 0` sentinel collision in `trace_decision`

`TradeEventRecord` writer persists `decision_id` as `0` when `None` to satisfy a NOT NULL constraint; reader maps `0` back to `None`. Two collision modes: (1) `JournalQuery::trace_decision(0)` returns every NFR17-violating row across all sources, scrambled; (2) a legitimate `decision_id = 0` (first decision after a fresh signal-id-generator startup) round-trips as `None`, scrambling the audit trail. Asymmetric to `order_states.decision_id` which is nullable.

One-line guard in `trace_decision` closes both cases. See B-2.

### Replay journal pollution risk

Pre-7.4, replay records were indistinguishable from live records (a real defect 7.4 fixed). Now they're tagged `source = "replay"` — but they still write to the same default database (`data/journal.db`). `paper_readiness_report` correctly filters by source, but `trace_decision` does not, and the journal grows without bound across replay runs. An operator running 50 replays a day against `data/journal.db` is silently bloating the production audit trail.

Default replay to `data/replay.db` (a one-line wiring change in `ReplayOrchestrator::new`) closes this. See B-3.

## Action Items

### Process Improvements

**P-8. `MarketDataFeed` trait semantic gap is a pre-Epic-8 cleanup item.**
Replace `next_event() -> Option<MarketEvent>` with `next_event() -> NextEvent` where `NextEvent { Event(MarketEvent), Idle, Terminated }`. The current `Option::None` conflates "no event right now" with "stream ended" — fine for replay's bounded Parquet source, structurally wrong for live data. Pre-Epic-8 because Story 8.2's "wire Rithmic" path depends on the trait having the right shape.
**Why:** 7.3 dev notes' "one-line Epic-8 swap" claim is false without this. Story 8.2 cannot wire a `RithmicMarketDataFeed` cleanly while the trait conflates idle and EOF.
**Owner:** Pre-Epic-8 cleanup spec (B-1).
**Success:** `pump_until_idle` exits only on `Terminated`; `Idle` round-trips through the orchestrator without terminating the loop; replay still terminates correctly when the Parquet source returns `Terminated`.
**Deadline:** Apply before Story 8.2 spec is finalized.

**P-9. Story 8.2 scope discipline + parallel-orchestrator codification.**
Story 8.2 today inherits ~7 OPEN deferred items + 7-4 B-1 + parallel-orchestrator unification + the pre-Epic-6-cleanup forward-looking items. Architect-led decomposition before story creation. The decomposition must take a clear position on: (a) does Story 8.2 unify replay/paper/live onto the production `EventLoop`, OR (b) does the architecture doc codify three parallel orchestrators with explicit subsystem-coverage matrices? Not deciding is the worst answer — the parallel pattern has been silently growing for two epics.
**Why:** AC verification has been silently bypassed for Epic 7 ("breakers function normally in paper" is structurally unverified today). Without an architectural decision, Story 8.2 will either silently inherit the bypass or attempt a too-large unification mid-story.
**Owner:** Winston (Architect) reviews; story creator (Amelia / `bmad-create-story`) drafts.
**Success:** Epic 8 story-creation produces an explicit subsystem-coverage matrix (per orchestrator, which subsystems run / are stubbed / are absent) in the architecture document or in the Epic 8 epics file.
**Deadline:** Apply at Epic 8 story creation, before Story 8.2 spec freezes.

**P-10. AC verification gap pattern — "mechanism exists, production wiring does not."**
Three of four 7-x reviews flagged this. Future Story 8.x ACs of form "X happens end-to-end" must require an integration test that exercises the production wiring path — i.e. a test that constructs the same object graph the binary constructs, drives the same event stream, and asserts on the same observable artifact. Tests that synthesize the data the production wiring doesn't yet provide (the 7-4 `record_fill` helper is the canonical antipattern) MUST be flagged at code review and either replaced or accompanied by an explicit "wiring gap" deferred-work entry.
**Why:** 7-4 B-1 surfaced because production paper writes zero paper-tagged rows despite 9 passing integration tests. The discipline gap is structural, not specific to 7.4.
**Owner:** Reviewer (whoever runs `bmad-code-review`).
**Success:** Every 8.x review either (a) verifies AC end-to-end against production wiring, or (b) explicitly catalogues the wiring gap with an owner.
**Deadline:** In effect immediately for Story 8.x reviews.

## Closure

Epic 7 delivered the determinism and source-tag mechanisms required to make replay verification and paper-trade auditing real: a three-tier determinism model with insta-snapshot regression, a `JournalSender::with_source` override that cleanly distinguishes paper / replay / live records, and an idempotent schema migration that handles pre-7.4 journals correctly. The fixed-point P&L tier captures raw `i64` to dodge `FixedPrice`'s lossy serde, and `Signal::reset()` is now demonstrably load-bearing (not just a docstring contract).

Four stories shipped over two days, preceded by the routine pre-epic cleanup commit that has now become the project's standard rhythm. Zero BLOCKING findings on three of four stories; one BLOCKING on 7.4 (the production paper-mode wiring gap), correctly scoped as Story 8.2's deliverable rather than as a Story 7.4 defect.

The growth area is structural: three parallel orchestrators (replay, paper, future-live) each skip the production `EventLoop`, bypassing all of the safety subsystems Epics 4–6 built. Integration tests "stitch the data the production wiring doesn't yet provide" (the 7-4 `record_fill` helper is the canonical instance). This pattern has been growing silently for two epics; Epic 8 must take a position on it rather than continuing to inherit it.

P-8 (`MarketDataFeed` semantic gap) closes via a small pre-Epic-8 cleanup commit. P-9 (Story 8.2 scope discipline + parallel-orchestrator codification) is the architectural decision that gates Story 8.2's clean start. P-10 (AC verification gap pattern) is the discipline change for Story 8.x reviews.

Carry forward into Epic 8: tight component discipline (held), tight integration discipline (now requires architect-level scope discipline because the deferred-list is too large to inherit silently), structural-invariant-over-docstring still the right principle (and now extended to "structural-test-over-mechanism-claim" — see P-10), plus a small pre-Epic-8 prep commit to close the API-shape items before Story 8.1 / 8.2 begin.
